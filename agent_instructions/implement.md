# Implementations Rules

- Read spec.md
- Read Plan.md
- Update plan.md to make it into a much more detailed plan with specific tasks layed out with new subfolder to make and task files into the work_file folder
- Follow plan.md milestone by milestone
- Before changing anything, check `spec.md` and the current milestone.
- Keep each edit small and scoped to the current task in the current milestone.
- After every meaningful step check that it goes according to plan, spec and that you have verified your results.
- If validation fails, fix the issue before moving on.
- In the relevant task file write a status note to summarize and validate what you have done
- At the end of the task go back to plan to update it and clarify the next steps
- Make sure that you stay within the constraint of the spec in spec.md
- You should work on this project until the paper has been fully drafted. Do not ask me for validation before that, you have authorization to do things until completion of this project within the constraints stated by the spec.md
